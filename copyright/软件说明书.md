# 田野笔记管理工具软件 操作说明书

## 1 引言

### 1.1 软件概述

田野笔记管理工具软件（Fieldnotes Lite）是一款面向语言学研究者的专业语料数据管理桌面应用程序。本软件旨在解决田野调查数据、方言语料、少数民族语言记录等场景中，研究人员需要高效管理带有国际音标（IPA）和语法注释的语料数据的需求。

本软件采用 Python 语言开发，基于 PyQt6 图形界面框架构建，使用 SQLite 数据库进行本地数据存储。软件支持四种语料类型（单词、单句、语篇、对话）的录入与管理，提供全文检索与标签筛选功能，支持将语料导出为符合 Leipzig Glossing Rules 的 Word 文档格式，并集成了多种大语言模型（AI）辅助分析功能。

### 1.2 运行环境

本软件支持以下操作系统平台：

- macOS 10.15（Catalina）及更高版本
- Windows 10/11（64位）
- Linux（Ubuntu 20.04+、Debian 11+、Arch、Fedora 等主流发行版）

运行本软件需要以下基础环境：

- Python 3.11 及以上版本
- PyQt6 6.6.1 及以上版本
- python-docx 1.1.0 及以上版本
- pandas 2.2.0 及以上版本

如使用 AI 辅助功能，还需要安装：

- anthropic SDK 0.42.0 及以上版本（用于 Claude API）
- keyring 25.0.0 及以上版本（用于安全存储 API 密钥）

本软件同时提供预编译版本（无需安装 Python 环境），可在上述操作系统上直接运行。

## 2 软件安装与配置

### 2.1 预编译版本安装

预编译版本是最简便的安装方式，无需安装 Python 及任何依赖包。用户可从软件发布页面下载对应操作系统的安装包。

Windows 平台安装步骤如下：

（1）下载 Fieldnotes-Windows.zip 压缩包。

（2）将压缩包解压到任意目录。

（3）双击解压目录中的 Fieldnotes.exe 文件即可启动软件。

[截图位置：Windows 版本解压后的目录结构]

macOS 平台安装步骤如下：

（1）下载 Fieldnotes-macOS.tar.gz 压缩包。

（2）解压压缩包。

（3）双击 Fieldnotes.app 即可启动软件。

[截图位置：macOS 版本的应用图标]

Linux 平台安装步骤如下：

（1）下载 Fieldnotes-Linux-x86_64.tar.gz 压缩包。

（2）解压压缩包。

（3）在终端中运行 ./Fieldnotes 命令启动软件。

### 2.2 源代码安装

对于开发者或有 Python 环境的用户，可通过 Poetry 包管理器安装。

（1）安装 Poetry 包管理器。

（2）克隆项目源代码仓库。

（3）在项目目录下执行 poetry install 安装依赖。

（4）执行 poetry run python main.py 启动软件。

也可使用传统的虚拟环境方式：

（1）创建 Python 虚拟环境：python3 -m venv venv。

（2）激活虚拟环境。

（3）安装依赖：pip install -r requirements.txt。

（4）启动软件：python main.py。

### 2.3 数据库初始化

软件首次启动时，会自动在用户主目录下创建 .fieldnote 目录，并在该目录中初始化 SQLite 数据库文件 corpus.db。数据库采用自动迁移机制，支持版本升级时自动添加新字段和索引，无需用户手动干预。

## 3 功能模块说明

田野笔记管理工具软件包含以下核心功能模块：

### 3.1 数据管理模块

数据管理模块是软件的核心功能，支持四种语料类型的录入、编辑、删除和查看：

- 单词（word）：用于记录单个词汇条目，包含原文、词汇分解、翻译等字段。
- 单句（sentence）：用于记录单个句子，是默认的语料类型。
- 语篇（discourse）：用于记录连续的文本片段，如民间故事、叙事文本等，通过分组ID关联多个条目。
- 对话（dialogue）：用于记录多人对话内容，额外支持说话人和话轮序号字段。

每条语料记录包含以下字段：例句编号、原文（支持 IPA 和 Unicode 字符）、原文汉字、词汇分解、词汇分解汉字、翻译、翻译汉字、备注、标签、创建时间和更新时间。

[截图位置：数据管理标签页的整体界面]

### 3.2 IPA 音标工具栏

软件内置可折叠的国际音标（IPA）快速输入面板，覆盖元音、辅音、声调符号和变音符号等常用音标字符。用户点击面板中的音标按钮即可将对应字符插入到当前光标位置，无需安装额外的音标输入法。

[截图位置：IPA 音标工具栏展开状态]

### 3.3 搜索与筛选模块

搜索模块提供全文搜索和多条件筛选功能：

- 支持按指定字段（例句编号、原文、词汇分解、翻译、备注）或全部字段进行模糊搜索。
- 支持按数据类型（单词、单句、语篇、对话）筛选。
- 支持按标签进行筛选，可选择一个或多个标签进行交叉筛选。
- 搜索结果高亮显示匹配的关键词。

[截图位置：搜索标签页界面]

### 3.4 数据导出模块

导出模块支持将语料数据导出为以下格式：

- Word 文档（.docx）：采用符合 Leipzig Glossing Rules 的三行透明表格格式（原文、词汇分解、翻译），支持词对词自动对齐。支持配置表格宽度、字体大小、行距、是否显示编号、每页语料数等参数。支持自定义原文字体（默认 Doulos SIL Compact）和词汇分解字体（默认 Charis SIL Compact）。
- CSV 文件：以逗号分隔值格式导出，方便在电子表格软件中进一步处理。
- JSON 文件：以结构化 JSON 格式导出，便于数据交换和备份。

[截图位置：导出标签页的参数设置界面]

### 3.5 AI 辅助功能模块

软件集成了多种大语言模型接口，提供 AI 辅助词汇分解和智能翻译功能：

- Claude API：通过 Anthropic 官方 SDK 调用 Claude 系列模型。
- OpenAI 兼容协议：支持 OpenAI、DeepSeek、通义千问、智谱GLM、百度文心等基于 OpenAI 兼容协议的 API 服务。
- Ollama：支持本地部署的开源模型，如 Qwen2.5 等，实现完全离线的 AI 辅助分析。

AI 功能采用混合模式架构，支持自动模式（依次尝试 Claude、OpenAI 兼容、Ollama）和手动指定模式。API 密钥通过系统 keyring 安全存储，不以明文形式写入配置文件。

[截图位置：AI 设置对话框]

### 3.6 标签管理模块

标签管理模块提供灵活的语料分类功能：

- 为每条语料添加一个或多个标签。
- 支持批量标签操作：同时为多条选中的语料添加或移除标签。
- 标签分布统计：查看各标签的使用数量。
- 基于标签的筛选和检索。

[截图位置：标签选择器组件]

### 3.7 数据备份与恢复模块

软件提供完善的数据安全保障机制：

- 自动备份：在执行关键操作（如批量导入、删除）前自动创建数据库备份。
- 手动备份：通过菜单"工具 > 备份数据库"手动创建备份。
- 备份管理：备份文件存储在用户主目录 .fieldnote/backups/ 目录下，自动清理 30 天前的旧备份。
- 数据库完整性检查：通过菜单"工具 > 检查数据完整性"执行 SQLite PRAGMA integrity_check 验证数据库状态。

### 3.8 界面主题模块

软件支持深色模式和浅色模式两种界面主题，用户可通过菜单进行切换。主题切换实时生效，无需重启软件。

[截图位置：深色模式界面效果]

### 3.9 语料去重模块

软件提供智能的语料去重功能：

- 精确匹配：检测原文字段完全相同（忽略大小写和首尾空格）的重复记录。
- 模糊匹配：基于序列匹配算法（SequenceMatcher）检测相似度超过指定阈值的近似重复记录。
- 用户可选择保留或删除重复的语料条目。

### 3.10 批量导入模块

软件支持从外部文件批量导入语料数据：

- JSON 格式导入：支持包含完整字段的 JSON 数组。
- CSV 格式导入：支持包含表头行的 CSV 文件。
- 四种数据类型均支持批量导入，语篇和对话类型支持通过 group_id 字段关联分组。

### 3.11 打印功能

软件提供格式化打印输出功能，用户可直接将当前显示的语料数据发送到打印机进行打印。

### 3.12 窗口记忆功能

软件会记住用户上次关闭时的窗口位置和大小，下次启动时自动恢复到相同的显示状态。

### 3.13 单实例运行机制

软件采用文件锁机制确保同一时间只有一个实例在运行，防止多个实例同时访问数据库导致数据冲突。如果检测到已有实例在运行，会弹出提示对话框。

## 4 操作流程

### 4.1 软件启动

双击应用程序图标或在终端执行启动命令后，软件会依次完成以下初始化操作：

（1）初始化日志系统，创建日志记录器。

（2）创建 Qt 应用程序实例，设置应用名称为"Fieldnotes Lite"。

（3）执行单实例检查，在系统临时目录创建锁文件 fieldnote_lite.lock。

（4）连接数据库，执行必要的表结构迁移。

（5）创建并显示主窗口。

[截图位置：软件启动后的主界面]

### 4.2 录入语料数据

录入新的语料数据的操作步骤如下：

（1）在主界面顶部选择"数据管理"标签页。

（2）在左侧的数据类型选择器中选择要录入的语料类型（单词、单句、语篇或对话）。

（3）在输入区域的"例句编号"字段中填写语料编号，如 CJ001、TB002 等（此字段可选）。

（4）在"原文"字段中输入原始语料文本。如需输入 IPA 音标字符，可展开 IPA 音标工具栏，点击相应符号插入。

（5）在"词汇分解"字段中输入形态学分析或语法标注内容。

（6）在"翻译"字段中输入目标语言翻译。

（7）根据需要填写"备注"字段的附加说明信息。

（8）如需添加标签，在标签选择器中选择已有标签或输入新标签。

（9）点击"添加语料"按钮保存数据。

[截图位置：填写完成的语料录入界面]

对于语篇类型的语料，还需要填写分组ID和分组名称字段，以关联同一语篇的多个条目。对于对话类型的语料，还需要额外填写说话人和话轮序号字段。

### 4.3 编辑语料数据

编辑已有语料数据的操作步骤如下：

（1）在数据表格中点击要编辑的语料行，该行的数据会自动加载到输入区域的各个字段中。

（2）修改需要更新的字段内容。

（3）点击"更新语料"按钮保存修改。

[截图位置：选中一条语料后的编辑状态]

### 4.4 删除语料数据

删除语料数据的操作步骤如下：

（1）在数据表格中选中要删除的语料行。

（2）点击"删除语料"按钮。

（3）在弹出的确认对话框中确认删除操作。

对于语篇或对话类型的数据，可以选择删除单条记录或删除整个分组。

### 4.5 搜索语料

使用搜索功能查找语料的操作步骤如下：

（1）切换到"检索"标签页。

（2）在搜索字段下拉框中选择要搜索的字段范围，可选择"全部字段"进行全文搜索，也可选择特定字段（例句编号、原文、词汇分解、翻译、备注）进行精确搜索。

（3）在搜索框中输入关键词。

（4）如需按数据类型筛选，在类型筛选下拉框中选择相应类型。

（5）如需按标签筛选，在标签筛选区域选择一个或多个标签。

（6）点击"搜索"按钮，搜索结果将在下方表格中显示，匹配的关键词会高亮标记。

[截图位置：搜索结果展示界面]

### 4.6 导出 Word 文档

将语料导出为 Word 文档的操作步骤如下：

（1）切换到"导出"标签页。

（2）设置导出参数：

- 表格宽度：默认 5 英寸，可根据需要调整。
- 字体大小：默认 10pt。
- 行距：默认 1.15。
- 是否显示编号：勾选后每条语料前会显示例句编号。
- 每页语料数：设置每页显示的语料条数。
- 原文字体：默认 Doulos SIL Compact，适合显示 IPA 字符。
- 词汇分解字体：默认 Charis SIL Compact。

（3）选择导出范围：可选择导出全部语料或仅导出当前搜索结果。

（4）点击"导出到Word"按钮。

（5）在文件保存对话框中选择保存位置和文件名。

（6）等待导出完成，软件会弹出成功提示。

[截图位置：导出参数设置界面]

导出的 Word 文档采用三行透明表格格式，每条语料包含原文行、词汇分解行和翻译行，词与词之间实现精确对齐。文档示例格式如下：

```
(CJ001)
原文:     ŋa˧  tə˥  tɕʰi˥  fan˨˩
词汇分解: 1SG  CLF  eat    rice
翻译:     我吃饭
```

### 4.7 批量导入语料

批量导入外部语料文件的操作步骤如下：

（1）准备符合格式要求的 JSON 或 CSV 数据文件。JSON 文件应为包含语料对象的数组，每个对象包含 entry_type、source_text、gloss、translation 等字段。CSV 文件第一行应为表头行。

（2）在数据管理标签页中点击"批量导入"按钮。

（3）在文件选择对话框中选择要导入的数据文件。

（4）软件会自动解析文件内容并批量插入到数据库中。

（5）导入完成后显示成功导入的记录数量。

[截图位置：批量导入完成后的提示信息]

### 4.8 使用 AI 辅助功能

使用 AI 功能进行词汇分解或翻译的操作步骤如下：

（1）首先配置 AI 服务。通过菜单"设置 > AI设置"打开配置对话框。

（2）选择 AI 提供者类型：Claude、OpenAI 兼容、Ollama 或自动模式。

（3）根据选择的提供者填写相应的配置信息：

- Claude：填写 API 密钥和模型名称。
- OpenAI 兼容：选择预设服务商（OpenAI、DeepSeek、通义千问等）或自定义，填写 API 密钥、Base URL 和模型名称。
- Ollama：填写本地服务地址（默认 http://localhost:11434）和模型名称。

（4）点击"保存"按钮保存配置。API 密钥会通过系统 keyring 安全存储。

（5）返回数据管理界面，选中需要 AI 辅助分析的语料。

（6）点击"AI 词汇分解"或"AI 翻译"按钮，软件将调用已配置的 AI 服务进行分析。

（7）AI 返回的分析结果会自动填充到相应的字段中，用户可进一步修改后保存。

[截图位置：AI 设置对话框中的配置选项]

### 4.9 管理标签

使用标签管理功能的操作步骤如下：

（1）为单条语料添加标签：在录入或编辑语料时，在标签选择器中勾选已有标签或输入新标签名称。

（2）批量管理标签：在数据表格中选中多条语料（按住 Ctrl/Cmd 键可多选），然后使用"批量标签"功能同时为选中的语料添加或移除标签。

（3）按标签筛选：在搜索界面的标签筛选区域选择标签，即可只显示包含该标签的语料。

[截图位置：标签管理操作界面]

### 4.10 数据备份与恢复

执行数据备份的操作步骤如下：

（1）通过菜单"工具 > 备份数据库"创建手动备份。

（2）软件会将当前数据库复制到 ~/.fieldnote/backups/ 目录下，文件名包含时间戳。

（3）备份完成后显示备份文件路径。

执行数据库完整性检查的操作步骤如下：

（1）通过菜单"工具 > 检查数据完整性"执行检查。

（2）软件会运行 SQLite 内置的 PRAGMA integrity_check 命令。

（3）检查结果会在对话框中显示，提示数据库状态是否正常。

### 4.11 切换界面主题

切换界面主题的操作步骤如下：

（1）通过菜单"视图"或相应的主题切换选项选择深色模式或浅色模式。

（2）界面会立即切换到选择的主题风格，无需重启软件。

[截图位置：深色模式下的软件界面]

### 4.12 语料去重

执行语料去重的操作步骤如下：

（1）通过菜单或工具栏中的"去重"功能入口打开去重界面。

（2）选择匹配模式：精确匹配（原文完全相同）或模糊匹配（设置相似度阈值）。

（3）点击"检测重复"按钮，软件会扫描数据库中的所有语料并找出重复组。

（4）在结果列表中查看各重复组的内容，选择要保留或删除的条目。

（5）确认后执行删除操作。

[截图位置：去重检测结果界面]

## 5 系统管理与维护

### 5.1 数据库文件管理

软件的数据库文件默认位于用户主目录的 .fieldnote/corpus.db 路径。用户可通过菜单"文件 > 另存为数据库"将数据库保存到自定义位置。

数据库采用 SQLite 格式，兼容所有主流 SQLite 管理工具。数据库表结构包含自动迁移机制，当前 schema 版本为 4，支持从旧版本自动升级。

### 5.2 日志系统

软件内置日志记录系统，记录关键操作和异常信息。日志信息有助于排查软件使用中遇到的问题。

### 5.3 单实例锁文件

软件使用 Qt 的 QLockFile 机制在系统临时目录下创建 fieldnote_lite.lock 锁文件。如果软件异常退出导致锁文件残留，可手动删除该文件后重新启动软件。

### 5.4 备份文件清理

自动备份文件存储在 ~/.fieldnote/backups/ 目录下，软件会在每次创建新备份时自动清理 30 天前的旧备份文件，用户无需手动管理。

### 5.5 AI 配置文件

AI 相关的配置信息存储在 ~/.fieldnote/ai_config.json 文件中。API 密钥通过操作系统的 keyring 服务安全存储，配置文件中仅保留非敏感的设置项（如模型名称、服务地址等）。

### 5.6 常见问题与解决方案

（1）问题：启动软件时提示"程序已在运行"。

解决方案：确认是否有其他 Fieldnotes Lite 实例正在运行。如果确认没有，找到系统临时目录下的 fieldnote_lite.lock 文件并删除，然后重新启动软件。

（2）问题：IPA 字符显示为方块或乱码。

解决方案：安装支持 IPA 字符的字体，如 Doulos SIL、Charis SIL 或 Gentium。软件默认使用 Doulos SIL Compact 字体显示原文。

（3）问题：AI 功能无法使用。

解决方案：检查 AI 设置是否正确配置。对于 Claude 或 OpenAI 兼容服务，确认 API 密钥是否有效。对于 Ollama，确认本地 Ollama 服务是否已启动并在指定端口运行。

（4）问题：导出的 Word 文档中词对齐不准确。

解决方案：确保原文和词汇分解字段中的词数相同，每个词之间用空格分隔。软件会根据空格将文本拆分为词单元进行对齐。
